# 智能记账本产品需求文档（PRD）

> **版本**：V1.0 | **日期**：2026年4月 | **目标平台**：HarmonyOS NEXT


## 一、产品概述

### 1.1 产品定位

“简账”是一款面向年轻个人用户的智能记账应用。不同于传统记账工具的“手动录入”模式，产品以“轻量、智能、美学”为核心设计理念——让记账从一项需要意志力坚持的任务，转变为一种愉悦的自我管理仪式。

### 1.2 目标用户

- **核心用户**：18-30岁学生/初入职场的年轻人
- **用户画像**：Z世代、数字原生、审美敏感、抗拒手动记账、需要消费洞察但不希望操作复杂

### 1.3 产品愿景

**“让每一笔钱都记得清清楚楚，让每一次记账都赏心悦目”**


## 二、创新功能设计

> 以下功能体现本产品的差异化竞争力，部分已在行业前沿探索中

### 2.1 核心创新

| 功能 | 创新点 | 用户价值 |
|------|--------|----------|
| **AI智能记账** | 支持语音/截图/短信自动识别账单 | 3秒完成记账，零手动输入 |
| **消费热力图** | 日历视图结合支出强度色块 | 直观感知消费节奏 |
| **动态主题引擎** | 主题跟随心情/场景/节日自动切换 | 记账也有仪式感 |
| **预算挑战赛** | Gamification预算管理机制 | 让省钱变得有趣 |
| **智能消费洞察** | AI分析消费习惯并给出建议 | 从记账到理财的跃迁 |

### 2.2 各功能详细说明

#### 🔮 功能一：AI智能记账（行业前沿）
- **语音记账**：说出“今天午饭花了35块”，自动识别金额、类别、时间
- **截图记账**：支付成功页截图后，App自动识别并弹出记账确认
- **短信/通知解析**：银行扣费短信自动生成账单待确认
- **自动分类**：基于历史行为+商户关键词，智能匹配消费类别（准确率目标≥90%）

#### 🎨 功能二：动态主题引擎（美学创新）
- **8大主题皮肤**：静谧蓝·星辰账海、活力橙·金辉轨迹、火焰红·克制警示、神秘紫·资产沉淀等
- **主题自动切换**：可设定工作日/周末不同主题、晚间自动深色模式
- **节日限定主题**：春节“财神到”、圣诞“雪花银”等限时掉落
- **预算状态联动**：预算紧张时自动切换到警示主题配色

#### 🎮 功能三：预算挑战赛（游戏化）
- **月度挑战**：设定预算目标，连续达标解锁勋章
- **消费等级**：像游戏段位一样，根据记账完整度/预算控制率升级
- **社交轻竞技**（可选）：与好友PK本月省钱排行（隐私保护前提下）

#### 📊 功能四：智能消费洞察
- **月度AI报告**：分析消费结构变化，如“本月餐饮支出较上月增加15%，主要集中在外卖”
- **省钱建议**：识别可优化的消费习惯，如“每周3杯奶茶，换成每周2杯可省XX元”
- **异常提醒**：单笔大额/某类别超支时主动推送提醒


## 三、核心功能模块

### 3.1 功能架构

```
简账App
├── 首页（记账核心）
│   ├── 快捷记账入口（大+号按钮）
│   ├── 当日/当月收支概览
│   ├── 最近账单列表
│   └── 快捷卡片（可自定义）
├── 统计报表
│   ├── 消费趋势图（折线图）
│   ├── 分类占比图（饼图/条形图）
│   ├── 消费热力图
│   └── AI月报
├── 预算中心
│   ├── 月度预算设置
│   ├── 预算进度条
│   ├── 挑战赛入口
│   └── 勋章墙
└── 个人中心
    ├── 账本管理（多账本）
    ├── 主题设置
    ├── 数据备份/恢复
    └── 隐私设置
```

### 3.2 核心页面详细说明

#### 首页
- **顶部**：展示当月总支出、总收入、结余
- **中部**：大号“+”记账按钮，支持长按语音记账
- **下部**：按日期分组的账单列表
  - 每天显示：日期、星期、当日总支出、当日总收入
  - 每条账单：分类图标、商户/备注、金额、时间
  - 左滑编辑/删除，右滑复制

#### 记账页
- 支出/收入切换（顶部Tab）
- **金额输入**：大数字键盘，支持计算器式输入
- **分类选择**：九宫格图标，支持自定义分类
- **日期/时间**：默认当前，可修改
- **备注**：支持常用备注模板
- **拍照**：可上传小票/截图（可选）
- **保存**：一键完成，返回首页并刷新

#### 统计页
- **时间筛选**：周/月/年/自定义
- **趋势图**：柱状图展示每日/月收支对比
- **分类排行**：饼图展示支出TOP5
- **热力图**：日历视图，颜色越深支出越高
- **AI报告入口**：生成月度消费洞察PDF


## 四、UI/UX设计要求

### 4.1 整体设计风格

| 维度 | 要求 |
|------|------|
| **设计语言** | 现代极简主义 + 微质感（Neumorphism轻点缀） |
| **色彩体系** | 主色调建议低饱和度色系（如莫兰迪色），支出用橙/红，收入用绿/蓝 |
| **圆角规范** | 卡片大圆角（24px），按钮中圆角（12px） |
| **留白原则** | 充足的呼吸空间，信息密度适中 |
| **图标风格** | 线性图标为主，选中态填充，风格统一 |

### 4.2 交互体验要求

- **流畅度**：页面切换60fps，记账操作3步内完成
- **反馈**：点击有微动效，记账成功有轻触感反馈
- **键盘**：数字键盘触感反馈，支持计算器式连续输入
- **手势**：支持下拉刷新、侧滑返回、长按编辑等标准手势
- **无障碍**：支持系统字体大小跟随、读屏软件适配

### 4.3 视觉一致性要求

- 所有页面遵循8px网格对齐
- 字体层级清晰：标题18sp/粗体、正文14sp/常规、辅助12sp
- 深色模式完整适配，所有颜色定义亮色/暗色两套


## 五、技术需求

### 5.1 平台要求
- **首发平台**：HarmonyOS NEXT
- **架构模式**：Stage模型 + 声明式UI（ArkUI）
- **最低版本**：HarmonyOS 5.0及以上

### 5.2 数据存储
- 账单数据本地存储（首选ArkData关系型数据库）
- 支持云备份（华为帐号同步，可选）
- 导出格式：CSV、PDF

### 5.3 图表组件
- 使用开源图表库（如@ohos/mpchart）实现饼图/柱状图/折线图
- 日历热力图组件需自研或二次封装

### 5.4 性能指标
- 冷启动时间 < 1.5秒
- 数据库查询（月账单）< 200ms
- 图表渲染 < 300ms
- 包体积 < 30MB（首版）

### 5.5 安全要求
- 本地数据库加密存储
- 无必要不上传用户数据，云同步需用户明确授权
- 不接入广告SDK（避免影响体验）
